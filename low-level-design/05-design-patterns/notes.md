# Design Patterns

## Creational (how objects are created)
- Singleton: only one instance exists (e.g. config, logger)
- Factory: create objects without specifying exact class
- Builder: construct complex objects step by step

## Structural (how objects are composed)
- Adapter: make incompatible interfaces work together
- Decorator: add behavior without changing the class
- Composite: tree structure (e.g. file system)

## Behavioral (how objects communicate)
- Observer: one object notifies many (e.g. event listeners)
- Strategy: swap algorithms at runtime
- Command: encapsulate a request as an object
- State: change behavior based on internal state
