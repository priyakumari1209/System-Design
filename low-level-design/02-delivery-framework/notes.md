# Delivery Framework

## The 4-Step Framework (use in every interview)
1. Gather Requirements
2. Define Core Entities (classes/objects)
3. Define APIs / Methods on each entity
4. Write the Code

## Step 1 — Requirements
- Ask clarifying questions
- Functional requirements (what it must do)
- Non-functional (scale, concurrency?)

## Step 2 — Core Entities
- List all nouns from the problem
- Each noun = potential class

## Step 3 — APIs
- For each class: what methods does the outside world need?
- Keep APIs small and focused

## Step 4 — Code
- Start with the simplest class
- Add state (fields) + behavior (methods)
- Apply: "rules live with the entity that owns the data"
