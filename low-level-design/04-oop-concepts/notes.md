# OOP Concepts

## 4 Pillars
1. Encapsulation — hide internal state, expose only what's needed
2. Abstraction — show only relevant details
3. Inheritance — child class gets parent's properties/methods
4. Polymorphism — same method, different behavior per class

## Classes vs Objects
- Class = blueprint
- Object = instance of the blueprint

## Abstract Class vs Interface
- Abstract class: can have some implementation
- Interface: only method signatures (contract)

## When to use each?
- Interface: unrelated classes sharing a behavior (Flyable, Serializable)
- Abstract class: related classes sharing base code
